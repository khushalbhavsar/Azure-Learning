# Day 1: Understanding Cloud Concepts, Vocabulary and Terminology 

---

## ☁️ What is Cloud?

**Cloud** refers to a network of remote servers hosted on the internet that store, manage, and process data instead of using a local computer or personal server.

---

## ☁️ What is Cloud Computing?

**Cloud Computing** is the delivery of computing services (servers, storage, databases, networking, software, etc.) over the internet on a pay-as-you-go basis.

---

## 🔄 Difference: Public vs Private vs Hybrid Cloud

| Type              | Definition                                                                            | Example                              |
| ----------------- | ------------------------------------------------------------------------------------- | ------------------------------------ |
| **Public Cloud**  | Services are owned and operated by third-party providers and shared over the internet | Amazon Web Services, Microsoft Azure |
| **Private Cloud** | Cloud infrastructure is used exclusively by one organization                          | Company internal data center         |
| **Hybrid Cloud**  | Combination of public + private cloud, allowing data & apps to be shared              | On-prem + AWS setup                  |

---

## 📘 Vocabulary (Important Terms)

### 1. Virtualization

Technology that allows creating multiple virtual environments (servers, OS) on a single physical machine.

---

### 2. Virtual Machine (VM)

A software-based computer that runs an operating system and applications like a physical computer.

---

### 3. API (Application Programming Interface)

A set of rules that allows different software applications to communicate with each other.

---

### 4. Regions

Geographical locations where cloud providers have data centers (e.g., Mumbai region in AWS).

---

### 5. Availability Zones

Separate data centers within a region designed for fault isolation and high availability.

---

### 6. Scalability

Ability to increase or decrease resources based on demand.

---

### 7. Elasticity

Automatic scaling of resources up or down in real-time depending on workload.

---

### 8. Agility

Ability to quickly develop, test, and deploy applications using cloud resources.

---

### 9. High Availability

System design that ensures services remain operational with minimal downtime.

---

### 10. Fault Tolerance

Ability of a system to continue working even if some components fail.

---

### 11. Disaster Recovery

Process of restoring data and systems after a failure (backup & recovery strategies).

---

### 12. Load Balancing

Distributing incoming traffic across multiple servers to ensure no single server is overloaded.

---

## 🚀 Pro Tip (Interview Ready Summary)

* **Cloud = Infrastructure**
* **Cloud Computing = Service delivery over internet**
* **Scalability ≠ Elasticity** → Scalability is manual/plan-based, Elasticity is automatic
* **High Availability vs Fault Tolerance** → HA reduces downtime, FT avoids downtime
